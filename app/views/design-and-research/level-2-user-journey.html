How the LOA2 user journey works

This is a typical LOA2 user journey. However, each identity provider verifies identities in different ways. This depends on the evidence the user can provide and the methods supported by the identity provider.

    Service

  1. Whilst using service, get asked to prove identity

    2. Verify hub

  2. Say whether you've used Verify before

      If used Verify before, sign in

    1. Select identity provider to sign in with

    2. Sign in with username or email and password

    3. Receive two factor authentication code

    4. Enter two factor authentication code

    5. Confirm

    6. Redirect to service

    7. Continue using service

  3. Read about what GOV.UK Verify is

  4. Read about the identity providers

  5. Read how long it will take, and that you'll need to create an account

  6. Finding the right company for you

  7. Are you over 20? Have you lived in the UK for over x years?

  8. Do you have a passport or driving license?

  9. Do you have a mobile phone? Can you download apps?

  10. Choose an identity provider

  11. Read that you're about to go to the identity provider

    12. Identity provider: create an account

  12. Create a username or email and password

  13. Receive a confirmation code by email

  14. Check email for confirmation code

  15. Enter confirmation code

    16. Identity provider: two factor authentication

  16. Add phone number

  17. Choose two factor authentication method

  18. Receive authentication code

  19. Enter authentication code

    20. Identity provider: add personal details

  20. Add name, age and gender

  21. Add and confirm address

  22. Confirm all details

  23. Details are checked by the identity provider

    24. Identity provider: document checking

  24. Add document details

    1. Download app

    2. Take photo of the document

    3. Take selfie

  25. Add second document details

  26. Add card or bank details

    27. Identity provider: knowledge-based verification

  27. Knowledge-based verification question 1

  28. Knowledge-based verification question 2

      Knowledge-based verification unsuccessful

    1. Verify has failed to verify you

    2. Other ways to use this service

    29. Confirmation: from the identity provider back to the Verify Hub

  29. Successfully verified

  30. Confirm

  31. Re-direct to service

  32. Continue using service
