How the LOA 2 user journey works

This is a typical user journey for a level of assurance (LOA) 2 service. It’s different to the user journey for an LOA 1 service.

Before they sign up to GOV.UK Verify, users will be asked some questions to help them choose the right certified company.

They’ll usually need at least 2 documents to create an account with a certified company. These should be from:

  • a government department, for example a passport
  • a bank or building society, for example a credit or debit card

It typically takes about 10 to 15 minutes to create a GOV.UK Verify account, and less than a minute to sign in after that point.

    Service

  1. User is asked to prove their identity to use the service

    2. GOV.UK Verify hub

  2. User says if they've used GOV.UK Verify before

      Sign in (returning user journey)

      It takes less than a minute to sign in.

    1. User picks a certified company to sign in with

    2. User signs in with username or email and password

    3. User receives 2 factor authentication code

    4. User enters 2 factor authentication code

    5. User continues to use the service

    3. Sign up (new user journey)

    It takes about 15 minutes to create an account.

  3. User finds out what GOV.UK Verify is

  4. User finds out about certified companies

  5. User finds out how long it will take to verify themselves and how to create an account

  6. User chooses a certified company

  7. User is asked if they're over 20 and how long they've lived in the UK

  8. User is asked if they have a passport or driving licence

  9. User is asked if they have a mobile phone that can download apps

  10. User chooses a certified company

  11. User is told they need to go to the certified company’s website to create an account

    12. Certified company: create an account

  12. User creates a username or email and password

  13. User receives a confirmation code by email

  14. User checks email for confirmation code

  15. User enters confirmation code

    16. Certified company: set up 2 factor authentication

  16. User adds phone number or chooses an alternative 2 factor authentication method

  17. User receives authentication code

  18. User enters authentication code

    19. Certified company: add personal details

  19. User adds their name, age and gender

  20. User adds and confirms address

  21. User confirms all details

  22. User's details are checked by the certified company

    23. Certified company: documents are checked

  23. User enters document details

    1. User downloads the certified company's app

    2. User takes a photo of the document

    3. User takes a photo of themselves

  24. User adds second document details

  25. User adds their card or bank details

    26. Certified company: knowledge-based authentication

  26. User answers knowledge-based authentication question 1

  27. User answers knowledge-based authentication question 2

      If knowledge-based authentication is unsuccessful

    1. User is told they can't be verified

    2. User sees the other ways to use this service

    28. Confirmation: user is sent back to the GOV.UK Verify hub

  28. The certified company tells the user they've been successfully verified

  29. GOV.UK Verify confirms that they've been successfully verified

  30. User continues to use the service
