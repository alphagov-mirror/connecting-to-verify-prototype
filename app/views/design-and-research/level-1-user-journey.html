This is a typical user journey for a level of assurance (LOA) 1 service. Every certified company checks users' identities differently.

  3. User finds out what GOV.UK Verify is

  4. User finds out about certified companies

  5. User finds out how long it will take to verify themselves and how to create an account

  6. User chooses a certified company

  7. User finds out more about the certified company and the documents they'll need

    8. Certified company: create an account

  8. User creates a username or email and password

  9. User receives a confirmation code by email

  10. User checks email for confirmation code

  11. User enters confirmation code

    12. Certified company: set up 2 factor authentication

  12. User adds phone number or chooses alternative 2 factor authentication method

  13. User receives authentication code

  14. User enters authentication code

    15. Certified company: add personal details

  15. User adds their name, age and gender

  16. User adds and confirms address

  17. User confirms all details

  18. Their details are checked by the certified company

    19. Certified company: knowledge-based authentication

  19. User answers knowledge-based authentication question 1

  20. User answers knowledge-based authentication question 2

      If knowledge-based authentication is unsuccessful

    1. User is told they can't be verified

    2. User sees the other ways to use this service

    21. Confirmation: user is sent back to the GOV.UK Verify hub

  21. The certified company tells the user they've been successfully verified

  22. GOV.UK Verify confirms that they've been successfully verified

  23. User continues to use the service
