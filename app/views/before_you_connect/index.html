Build your service to work with GOV.UK Verify

You need to consider how you will build your service to connect to Verify.

Set up the Verify Service Provider

Verify uses SAML to securely exchange information about identities. The Verify Service Provider helps create SAML requests and translate responses from SAML to JSON, which is easier to read.

Download the Verify Service Provider

Decide if you need to use matching

Matching is a process where you make sure that a verified user is the same one as the one on your database. There are two different levels of matching you can use. Some services do not need matching at all.

Decide if you need matching
Service needs to link to existing data Service needs to keep users' data Level of matching needed
No No Not needed
No Yes Simple matching
Yes Yes Full matching
How to build a service with simple matching

Build a local matching service

The local matching service tries to match a user’s assured identity with a record in the government service’s data sources. This allows the government service to interact with the user. You need to build this yourself.

Download the local matching service

How to build a service with full matching

Build a local matching service

The local matching service tries to match a user’s assured identity with a record in the government service’s data sources. This allows the government service to interact with the user. You need to build this yourself.

Download the local matching service

Set up Matching Service Adapter

The matching service adapter converts SAML, the language to communicate with the Verify hub into JSON for your service and vice versa. It also performs encryption, decryption, and signing of SAML messages.

Download the local matching service
